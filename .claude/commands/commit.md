# /commit

*Karma Yoga — act, then release the fruit.*

Commit as soon as a task is complete. Do not accumulate.

Rules:
- One logical change per commit
- Commit message: imperative present tense, max 72 chars
- If the change is hard to describe in one line, it is probably two commits

Format:
```
<type>: <description>

[optional body — only if context adds value]
```

Types: `feat`, `fix`, `docs`, `refactor`, `chore`, `style`

The act of committing is the act of completion. It closes the loop.
An uncommitted change is unfinished work — a distraction, not an asset.

> *"Mā phaleṣu kadācana"* — Never be attached to the results. (Bhagavad Gītā, 2.47)
